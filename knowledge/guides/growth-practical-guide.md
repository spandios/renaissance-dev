## 📌 Week 1: 기초 세팅

### 목표: 측정할 수 있는 상태 만들기

### 1. GA4 세팅 (1시간)

```
필수 이벤트:
□ page_view (페이지 조회)
□ sign_up (회원가입)
□ login (로그인)
□ [핵심 액션] (우리 제품의 핵심 기능)
□ purchase / conversion (전환)
```

**GA4 빠른 세팅:**
```javascript
// gtag.js 기본
gtag('event', 'sign_up', {
  method: 'email'
});

gtag('event', '[핵심_액션]', {
  value: 1
});
```

### 2. AARRR 정의 (30분)

```
우리 제품의 AARRR:

Acquisition (획득): _______________
  → 측정: GA4 traffic source

Activation (활성화): _______________
  → 측정: [핵심 액션] 완료율

Retention (유지): _______________
  → 측정: D7 재방문율

Revenue (수익): _______________
  → 측정: 전환율, ARPU

Referral (추천): _______________
  → 측정: 공유/초대 수
```

### 3. North Star Metric 정하기 (30분)

**작은 팀을 위한 NSM 선택 기준:**
- 매주 측정 가능
- 팀 전체 (나 포함)가 영향 줄 수 있음
- 이게 오르면 제품이 성공하는 것

**예시:**
- SaaS: 주간 활성 사용자 (WAU)
- 이커머스: 주문 수
- 콘텐츠: 콘텐츠 소비 시간

### 📋 Week 1 체크리스트
- [ ] GA4 설치 & 핵심 이벤트 세팅
- [ ] 우리 AARRR 5단계 정의
- [ ] NSM 1개 결정
- [ ] 간단한 대시보드 (GA4 또는 스프레드시트)

### 🗣️ 팀 공유용

```
📊 우리 제품 측정 시작!

🎯 North Star Metric: [___]
→ 이 숫자가 우리의 나침반

📈 AARRR 퍼널:
1. 획득: [어떻게 유입?]
2. 활성화: [첫 성공 경험?]
3. 유지: [왜 다시 오나?]
4. 수익: [어떻게 돈 버나?]
5. 추천: [왜 추천하나?]

📱 GA4 대시보드: [링크]
```

## 📌 Week 2: 데이터 기반 의사결정

### 목표: 숫자로 현실을 보고 개선점 찾기

### 1. 퍼널 분석

```
현재 퍼널 (GA4에서 확인):

[방문] 1000명 (100%)
   ↓ ___%
[회원가입] ___명
   ↓ ___%
[핵심 액션] ___명
   ↓ ___%
[재방문 D7] ___명

🚨 가장 큰 이탈: ___ → ___ 단계
```

### 2. Zero to One 마인드셋

**작은 팀의 강점 활용:**

```
❌ 대기업처럼 하지 마라:
- 완벽한 기능
- 긴 개발 주기
- 많은 리소스

✅ 스타트업처럼 하라:
- "쓰레기 MVP"도 OK
- 1주 단위 배포
- 직접 사용자와 대화
```

**PMF (Product-Market Fit) 체크:**
- 40% 이상이 "매우 실망할 것"이라 답하면 PMF
- 아직 PMF 전이면? → 기능 추가 X, 핵심 가치 집중

### 3. 가설 세우기

**가설 템플릿:**
```
[변경]을 하면,
[지표]가 [X%] 개선될 것이다.
왜냐하면 [근거] 때문이다.

예시:
"온보딩을 3단계에서 1단계로 줄이면,
활성화율이 20% 개선될 것이다.
왜냐하면 현재 2단계에서 60%가 이탈하기 때문이다."
```

**ICE Score로 우선순위:**
- Impact (영향): 1-10
- Confidence (확신): 1-10
- Ease (쉬움): 1-10
- Score = (I + C + E) / 3

### 📋 Week 2 체크리스트
- [ ] GA4에서 퍼널 전환율 확인
- [ ] 가장 큰 이탈 지점 파악
- [ ] 가설 3-5개 작성
- [ ] ICE Score로 1순위 선정

### 🗣️ 팀 공유용

```
📊 이번 주 데이터 인사이트

🔍 퍼널 분석 결과:
- 가장 큰 문제: [___] 단계 (이탈 __%)
- 기회: [___]를 개선하면 [___]% 성장 가능

💡 이번 주 개선 가설:
"[변경]하면 [지표]가 [X%] 오를 것"

→ 다음 주 실험 시작!
```

## 📌 Week 3: 빠른 실험

### 목표: 1주 안에 가설 검증하기

### 1. Agile 미니 스프린트 (작은 팀용)

```
월요일: 계획 (30분)
- 이번 주 실험 1개 선정
- 성공 기준 정의
- 담당 배정

화-목: 실행
- 개발 & 배포
- A/B 테스트 시작

금요일: 회고 (30분)
- 결과 확인
- 배운 것
- 다음 주 계획
```

### 2. A/B 테스트 (간단 버전)

**작은 팀을 위한 A/B:**
```
□ 복잡한 도구 필요 없음
□ Feature flag로 50/50 분할
□ 1주일 데이터 수집
□ GA4 이벤트로 측정

// 간단한 Feature Flag
const variant = userId % 2 === 0 ? 'A' : 'B';

if (variant === 'B') {
  // 새 버전
}

// GA4로 측정
gtag('event', 'experiment_view', {
  experiment_name: 'onboarding_v2',
  variant: variant
});
```

### 3. MVP 마인드셋

**"완벽"보다 "빠른 학습":**

```
❌ 2주 개발 → 배포 → 실패 발견
✅ 3일 MVP → 배포 → 빠른 학습 → 개선

MVP 체크리스트:
□ 핵심 가치만 있나?
□ 3일 안에 만들 수 있나?
□ 측정할 수 있나?
```

### 📋 Week 3 체크리스트
- [ ] 1순위 가설로 실험 설계
- [ ] 1주 스프린트 실행
- [ ] A/B 테스트 (또는 before/after)
- [ ] 금요일 회고

### 🗣️ 팀 공유용

```
🧪 이번 주 실험

실험: [실험명]
가설: [___]하면 [___]가 [X%] 오를 것

결과:
- A (기존): ___%
- B (신규): ___%

📚 배운 것:
[핵심 인사이트]

→ 다음 실험: [___]
```

## 📌 Week 4: 지속 성장 시스템

### 목표: 혼자서도 지속 가능한 성장 루틴 만들기

### 1. Growth Loop 설계

```
우리 제품의 Growth Loop:

[신규 유저] 
    ↓
[가치 경험] ← 여기가 핵심!
    ↓
[자연스러운 공유/노출]
    ↓
[신규 유저 유입]
    ↓
(반복)
```

**작은 팀의 Growth Loop 예시:**
- 사용자가 만든 결과물 공유 → 링크 클릭 → 신규 가입
- 초대 기능 → 친구 가입 → 둘 다 혜택

### 2. 주간 Growth 루틴 (혼자서도 가능)

```
📅 매주 금요일 30분

1. 숫자 체크 (10분)
   - NSM 변화
   - 퍼널 전환율
   - 이번 주 실험 결과

2. 회고 (10분)
   - 뭘 배웠나?
   - 뭘 계속할까?
   - 뭘 바꿀까?

3. 다음 주 계획 (10분)
   - 다음 실험은?
   - 우선순위는?
```

### 3. Zero to One → One to N

```
Phase 1: Zero to One (PMF 찾기)
□ 핵심 가치 검증
□ 10명의 열성 사용자
□ "없으면 안 되는" 제품

Phase 2: One to N (스케일)
□ Growth Loop 작동
□ 반복 가능한 획득 채널
□ 자동화된 온보딩
```
