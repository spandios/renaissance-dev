# 🍽️ 점심 지식 Deep Dive: "Anyone can cook" - AI 시대의 코딩

**날짜**: 2026-02-05  
**소스**: [Lenny's Newsletter - Guillermo Rauch (Vercel CEO)](https://www.lennysnewsletter.com/p/anyone-can-cook-how-v0-is-bringing)

---

## 1. The Big Idea (핵심 관통하기)

> **"코딩은 이제 요리처럼 누구나 할 수 있는 일이 된다."**

Vercel CEO가 시연하는 v0의 진화: 프로토타입 도구 → **완전한 개발 환경**  
비개발자가 Git 워크플로우(브랜치, PR, 배포)를 사용해 프로덕션 코드를 기여하는 시대가 왔다.

---

## 2. Argument Breakdown (논리 해부)

**v0가 해결한 문제:**

❌ 기존 AI 코딩 도구:
• 프로토타입만 가능
• 프로덕션 배포는 개발자 몫
• Git 워크플로우 없음

✅ v0 + Git 통합:
• 브랜치 생성 → 코드 작성 → 프리뷰 → PR 제출
• 비개발자도 프로덕션 기여
• 개발 환경 설정 불필요

**실제 사례: skills.sh**
• Vercel이 v0로 직접 만든 AI 스킬 마켓플레이스
• 34,000개 커뮤니티 스킬, **시간당 500개 제출**
• 디자이너/PM도 기능 추가 가능

**핵심 워크플로우:**
1. v0에서 브랜치 생성
2. AI로 기능 구현 (예: 평점 시스템)
3. 브랜치 프리뷰로 테스트
4. PR 제출 → 리뷰 → 머지

> "마치 요리책을 보고 요리하듯, 누구나 코드를 기여할 수 있다."

---

## 3. Why It Matters (왜 중요한가)

**개발자에게:**
• 코드 작성 속도 ↑ (프로덕션 고려사항 포함)
• 반복 작업 자동화
• **아키텍처/리뷰에 집중** 가능

**조직에게:**
• 디자이너/PM이 직접 코드 기여 → 병목 해소
• Git 워크플로우 = 품질 관리 유지
• 개발 문화 민주화

**산업에:**
• 소프트웨어 생산성 혁명
• "코딩 능력" 진입 장벽 붕괴
• **설계/제품 감각이 더 중요**해짐

---

## 4. 💼 For Engineers (개발자에게)

### 위협인가, 기회인가?

**위협으로 보는 시각:**
• "비개발자도 코드 쓰면 내 가치는?"
• 코드 품질 우려

**기회로 보는 시각:**
• 단순 작업에서 해방 → 고차원 문제에 집중
• 팀 전체의 생산성 ↑ → 더 많은 기능, 더 빠른 성장
• **리뷰/아키텍처 역량이 핵심**으로 부상

### 실전 대응 전략

1. **v0 같은 도구 적극 활용**
   - 거부하지 말고 마스터하기
   - 프로덕션 코드도 AI로 초안 작성

2. **Git 워크플로우 강화**
   - PR 리뷰 역량 키우기
   - 코드 품질 표준 수립

3. **제품 감각 + 아키텍처**
   - "무엇을 만들까?"가 "어떻게 만들까?"보다 중요
   - 시스템 설계, 확장성, 보안 고려

4. **팀 협업 문화**
   - 비개발자 기여 환영
   - 페어 프로그래밍 → 페어 AI 코딩

---

## 5. 🍳 "Anyone can cook"의 의미

라따뚜이의 명대사처럼:
> "누구나 요리할 수 있다. 하지만 위대한 요리사는 어디서든 나올 수 있다."

AI 시대:
> "누구나 코딩할 수 있다. 하지만 위대한 소프트웨어는 설계에서 나온다."

**코드 작성 = commodity (일용품)**  
**제품 설계 = 차별화 요소**

---

## 💡 Key Takeaways

1. **v0 = Git + AI** → 비개발자도 프로덕션 기여
2. **Vercel 내부**에서도 프로덕션에 사용 중 (입증된 도구)
3. **개발자의 역할 변화**: 코드 작성 → 리뷰/설계
4. **르네상스 개발자 시대**: 기술 + 제품 + 협업
