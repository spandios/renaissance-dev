## 💡 핵심 개념: 왜 측정이 먼저인가?

### "측정하지 않으면, 개선할 수 없다"

```
❌ 감으로 하는 개발:
"이 기능 좋을 것 같아" → 2주 개발 → 배포 → 아무도 안 씀

✅ 데이터로 하는 개발:
숫자 확인 → 문제 발견 → 가설 → 실험 → 검증 → 개선
```

**작은 팀일수록 측정이 중요한 이유:**
- 리소스가 적으니 삽질할 여유 없음
- 숫자로 우선순위 정해야 함
- "느낌"으로 싸우지 않음

## 📊 GA4 세팅 (30분이면 충분)

### 필수 이벤트 5개

```javascript
// 1. 페이지 조회 (자동)
// GA4 기본 제공

// 2. 회원가입
gtag('event', 'sign_up', {
  method: 'email'  // 또는 'google', 'kakao' 등
});

// 3. 로그인
gtag('event', 'login', {
  method: 'email'
});

// 4. 핵심 액션 (우리 제품의 Aha! Moment)
gtag('event', 'core_action', {
  action_type: 'create_project',  // 예시
  value: 1
});

// 5. 전환 (구매, 구독 등)
gtag('event', 'purchase', {
  currency: 'KRW',
  value: 9900
});
```

### React/Next.js 예시

```typescript
// lib/analytics.ts
export const trackEvent = (
  eventName: string, 
  params?: Record<string, any>
) => {
  if (typeof window !== 'undefined' && window.gtag) {
    window.gtag('event', eventName, params);
  }
};

// 사용
trackEvent('sign_up', { method: 'kakao' });
trackEvent('core_action', { action_type: 'first_upload' });
```

## 🏴‍☠️ AARRR: 해적 지표 (Pirate Metrics)

### 각 단계가 의미하는 것

```
A - Acquisition (획득)
    "사람들이 어떻게 우리를 찾나?"
    → 측정: 트래픽 소스, 채널별 유입

A - Activation (활성화)  ← 가장 중요!
    "첫 방문에서 가치를 느꼈나?"
    → 측정: 핵심 액션 완료율

R - Retention (유지)
    "다시 돌아오나?"
    → 측정: D1, D7, D30 재방문율

R - Revenue (수익)
    "돈을 내나?"
    → 측정: 전환율, ARPU, LTV

R - Referral (추천)
    "남에게 알려주나?"
    → 측정: 공유 수, 초대 수, NPS
```

### 우리 제품에 대입하기 (예시)

```markdown
## 현재 프로젝트 앱의 AARRR

**Acquisition:** 앱스토어 검색, 인스타그램 광고
→ 측정: 설치 수, 채널별 CAC

**Activation:** 첫 사진 색감 보정 완료
→ 측정: 설치 후 24시간 내 보정 완료율

**Retention:** 일주일 내 재방문
→ 측정: D7 리텐션

**Revenue:** 프리미엄 구독
→ 측정: 구독 전환율

**Referral:** 보정 결과물 공유
→ 측정: 공유 버튼 클릭 수
```

### 왜 Activation이 가장 중요한가?

```
신규 유저 100명
    ↓
[Activation 10%] → 10명만 가치 경험
    ↓
[Retention 50%] → 5명 재방문
    ↓
결과: 95명 이탈

vs

신규 유저 100명
    ↓
[Activation 50%] → 50명이 가치 경험
    ↓
[Retention 50%] → 25명 재방문
    ↓
결과: 5배 차이!
```

**Activation을 높이면 전체 퍼널이 개선된다.**

## ⭐ North Star Metric (NSM)

### NSM = 팀의 나침반

**좋은 NSM의 조건:**
1. 매주 측정 가능
2. 우리가 직접 영향 줄 수 있음
3. 이게 오르면 제품이 성공하는 것

### 업종별 NSM 예시

| 유형 | NSM | 왜? |
|------|-----|-----|
| SaaS | 주간 활성 사용자 (WAU) | 활발히 사용 = 가치 제공 |
| 이커머스 | 주문 수 | 매출의 선행 지표 |
| 콘텐츠 | 총 시청 시간 | 몰입도 = 광고 수익 |
| 소셜 | 일간 게시물 수 | 콘텐츠 = 플랫폼 가치 |
| B2B | 고객당 활성 좌석 수 | 도입 = 이탈 방지 |

### NSM 정하는 프레임워크

```
1. "우리 제품이 성공하면 어떤 숫자가 올라가나?"
   → [후보 1], [후보 2], [후보 3]

2. "그 중 매주 측정 가능한 건?"
   → [후보 필터링]

3. "우리 팀이 직접 영향 줄 수 있는 건?"
   → [최종 선택]
```

### ⚠️ NSM 선정 시 주의

```
❌ 나쁜 NSM:
- MAU (한 달에 한 번 측정 → 반응 느림)
- 매출 (후행 지표, 영향 주기 어려움)
- 다운로드 수 (설치만 하고 안 쓰면 의미 없음)

✅ 좋은 NSM:
- WAU (Weekly Active Users)
- 주간 핵심 액션 수
- 주간 신규 활성 사용자
```
